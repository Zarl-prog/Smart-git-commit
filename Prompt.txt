## Skill Goal

## What Each Phase in SKILL.md Should Cover (in order)
1. Read CLAUDE.md / project rules first (override defaults if found)
2. Full diff analysis — categorize every changed file by concern
3. Security scan — run scan-secrets.sh, block commit if secrets found
4. Test gate — run detect-test-runner.sh, only commit if green
5. Atomic split decision — detect mixed concerns, plan multiple commits
6. Commit message construction — type, scope, subject, body (WHY), footers
7. Issue & ticket linking — GitHub issues, Jira, Linear auto-detection
8. Execute commits — selective staging, verify diff before each commit
9. Push strategy — never push to main, always feature branch
10. PR creation — use gh CLI with pr-template.md, default to draft
11. Release tagging — semver bump, CHANGELOG.md update, git tag + push

## Scripts Requirements

scan-secrets.sh should:
- Grep for: API keys, tokens, passwords, private keys, .env values, AWS keys
- Check for accidentally staged .env, .pem, .key, credentials files
- Output: CLEAN or list of suspicious lines with file:line references
- Exit code 0 = clean, 1 = secrets found (blocks commit)

detect-test-runner.sh should:
- Auto-detect: package.json (npm test), pytest.ini/pyproject.toml (pytest), 
  Cargo.toml (cargo test), go.mod (go test), Makefile (make test), 
  .rspec (rspec), mix.exs (mix test)
- Run the detected suite and capture pass/fail
- Output: PASS with test count, or FAIL with failing test names
- If no test runner found: warn user and ask if they want to proceed

generate-changelog.sh should:
- Read git log since last tag
- Group commits by type: Features, Bug Fixes, Performance, Security, Breaking Changes
- Format as Markdown with date and version
- Prepend to existing CHANGELOG.md or create new one

split-commits.sh should:
- Show git diff --stat grouped by directory/concern
- Suggest a split plan: "I see 3 concerns: auth (2 files), payments (1 file), tests (3 files)"
- Let user confirm or modify the plan
- Execute staged commits in the right order
